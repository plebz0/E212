Wymagania:
Zainstalowany Docker lub Docker Desktop
Wolny port 27017 na maszynie 

Uruchomienie MongoDB przy użyciu docker run:
W terminalu wpisz:
docker run -d --name birds -p 27017:27017 mongo:6.0
Opis parametrów:


Dane połączenia do bazy danych i porty:

Dane:
Host: localhost
Port: 27017
Nazwa kontenera birds (--name birds)
Nazwa bazy danych: birds
Login: brak
Hasło: brak
Wersja MongoDB: 6.0
Connection String: mongodb://localhost:27017

Porty:
Port lokalny: 27017
Port Docker: 27017
Mapowanie: -p 27017:27017:
 port lewy(27017) - port na hoście
 port prawy(27017) - port wewnątrz kontenera docker

Przykładowy adres połączenia używany w aplikacji:

MongoDB_URL = 'mongodb://localhost:27017'

Sprawdzenie, czy kontener działa:
docker ps

Jeżeli na liście widoczny jest kontener o nazwie „birds”, oznacza to, że MongoDB działa poprawnie.
Aplikacja po połączeniu się z bazą danych birds stworzy resztę sama. 

Zatrzymanie kontenera:
docker stop birds

Usunięcie kontenera:
docker rm birds
